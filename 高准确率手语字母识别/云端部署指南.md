## 🚀 部署选项

### Option 1: Streamlit Cloud（推荐）

Streamlit Cloud是最简单的部署方式，适合快速分享应用：

1. **准备GitHub仓库**
   - 将代码上传到GitHub仓库
   - 确保仓库包含以下文件：
     - `streamlit_app.py`
     - `requirements.txt`
     - 其他必要的Python文件
     - （如果有）预训练的`gesture_model.pkl`文件

2. **部署应用**
   - 访问 [Streamlit Cloud](https://share.streamlit.io/)
   - 注册/登录您的账户
   - 点击"New app"按钮
   - 选择您的GitHub仓库
   - 选择主分支
   - 在"Main file path"中输入`streamlit_app.py`
   - 点击"Deploy!"

3. **获取访问地址**
   - 部署完成后，Streamlit Cloud会生成一个公开的URL
   - 示例：`https://your-app-name.streamlit.app`
   - 所有用户都可以通过这个URL访问您的应用

### Option 2: Heroku

Heroku是另一个流行的部署平台：

1. **准备工作**
   - 确保项目中已包含以下文件（已为您创建）：
     - `requirements.txt`
     - `Procfile`
     - `setup.sh`

2. **安装Heroku CLI**
   - 下载并安装 [Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli)

3. **登录Heroku**
   ```bash
   heroku login
   ```

4. **创建Heroku应用**
   ```bash
   heroku create your-app-name
   ```

5. **部署应用**
   ```bash
   git push heroku master
   ```

6. **获取访问地址**
   - 部署完成后，访问 `https://your-app-name.herokuapp.com`

### Option 3: 使用Docker容器

使用Docker可以在任何支持容器的云服务上部署：

1. **准备工作**
   - 确保项目中包含`Dockerfile`（已为您创建）

2. **构建Docker镜像**
   ```bash
   docker build -t sign-language-recognizer .
   ```

3. **本地测试（可选）**
   ```bash
   docker run -p 8501:8501 sign-language-recognizer
   ```

4. **部署到云服务**
   - **阿里云容器服务**
   - **腾讯云容器服务**
   - **AWS Elastic Container Service (ECS)**
   - **Google Cloud Run**
   - **Azure Container Instances**

   根据您选择的云服务商，按照其文档进行部署。

## 🔧 部署前准备

在部署前，请确保：

1. **安装依赖已完整**
   - `requirements.txt`文件包含所有必要的依赖

2. **模型文件已准备**
   - 如果您有预训练的模型，请确保`gesture_model.pkl`文件已包含在项目中
   - 或者确保应用能自动生成或下载模型

3. **权限设置**
   - 确保脚本文件有执行权限：
     ```bash
     chmod +x setup.sh
     ```

## 🌐 访问应用

部署成功后，您将获得一个公开的URL，例如：
- Streamlit Cloud: `https://your-app-name.streamlit.app`
- Heroku: `https://your-app-name.herokuapp.com`
- 云服务器: `https://your-domain.com` 或 `http://your-server-ip:8501`

所有用户只需在浏览器中输入此URL，即可访问手语字母识别应用，无需安装任何软件。

## 📱 移动设备支持

部署到云端后，移动设备用户也可以通过浏览器访问应用，使用手机摄像头进行手势识别。

## ⚠️ 注意事项

1. **摄像头权限**：用户需要在浏览器中授予摄像头访问权限
2. **网络要求**：应用需要稳定的网络连接进行实时视频处理
3. **隐私考虑**：提醒用户应用仅在本地处理视频数据，不会上传到服务器（如适用）
4. **资源限制**：某些免费云服务可能有使用时间或资源限制
