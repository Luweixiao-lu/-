# 🔍 验证模型文件是否在GitHub上

## 问题

应用显示"模型未找到"，即使文件已经上传到GitHub。

## ✅ 验证步骤

### 1. 检查GitHub仓库

访问您的GitHub仓库，确认：
- ✅ `gesture_model.pkl` 文件存在
- ✅ 文件大小显示为约17MB
- ✅ 文件在仓库根目录（与 `app.py` 同级）

### 2. 检查文件路径

在GitHub上，文件应该位于：
```
您的仓库/
├── app.py
├── gesture_model.pkl  ← 应该在这里
├── requirements.txt
└── ...
```

### 3. 验证文件是否被Git跟踪

在本地运行：
```bash
git ls-files | grep gesture_model.pkl
```

如果命令有输出，说明文件已被Git跟踪。

## 🔧 如果文件不在GitHub上

### 方法1：强制添加大文件

```bash
# 1. 添加文件（即使很大）
git add -f gesture_model.pkl

# 2. 提交
git commit -m "Add model file"

# 3. 推送
git push
```

### 方法2：使用Git LFS（如果文件>100MB）

```bash
# 1. 安装Git LFS
brew install git-lfs  # macOS
# 或访问: https://git-lfs.github.com/

# 2. 初始化
git lfs install

# 3. 跟踪.pkl文件
git lfs track "*.pkl"

# 4. 添加配置
git add .gitattributes

# 5. 添加模型文件
git add gesture_model.pkl

# 6. 提交并推送
git commit -m "Add model with Git LFS"
git push
```

## ⚠️ 常见问题

### Q: 文件在GitHub上但应用还是找不到？

**A:** 可能的原因：
1. Streamlit Cloud还没有重新部署（等待2-5分钟）
2. 文件路径不对（检查是否在根目录）
3. 文件被.gitignore忽略（检查.gitignore）

### Q: 如何确认文件已推送？

**A:** 
```bash
# 检查远程仓库
git ls-remote --heads origin

# 查看远程文件
git ls-tree -r HEAD --name-only | grep gesture_model
```

### Q: 文件太大无法推送？

**A:** 使用Git LFS（见上面的方法2）
